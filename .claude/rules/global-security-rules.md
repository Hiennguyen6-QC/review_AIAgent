# Global Security & Interaction Rules

## 4. Work Process (Approval-First)
- For any architectural changes, file creations, or configuration edits, you MUST output a plan or proposal first and wait for explicit user confirmation before executing.

## 6. Output Naming Conventions
- **Requirement Analysis:** All analysis reports must follow the format `Analysis_<Ticket_ID_or_Function_Name>.md`.
- **Consistency:** Ensure that once a naming pattern is established for a project area, it is followed for all subsequent files in that category.

## 7. Mandatory Explanation for Advanced Tools & File Access
- **Explanatory First:** Before using the Browser tool, opening any local file path via URL, or performing ANY file edits (updating configs, rules, or code), the AI MUST provide a clear explanation to the user.
- **Why & What:** The explanation must include "Why" the action is needed and "What" specific file or address is being targeted.
- **Approval Rule:** After the explanation, the AI MUST wait for confirmation (e.g., "OK", "Go", or "Proceed") from the user before executing, unless it is part of a previously approved Implementation Plan.
